**Features**:
- ✅ Dio integration with shared instance from `dioProvider`
- ✅ Proper error handling using `ErrorHandle.handleDioException`
- ✅ Converts DioException to domain Failures

**API Endpoint**: `GET /data/dashboard`

### 2. **Repository Layer** 

#### Domain Repository (`dashboard_repository.dart`)

**Purpose**: Defines the contract for dashboard operations

```dart
abstract class DashboardRepository {
  Future<DashboardResponse> getDashboardData();
}
```

#### Data Repository (`dashboard_repository_impl.dart`)

**Purpose**: Implements the repository contract and orchestrates data sources

```dart
class DashboardRepositoryImpl implements DashboardRepository {
  final DashboardRemoteDataSource _remoteDataSource;
  
  @override
  Future<DashboardResponse> getDashboardData() async {
    // Future enhancement: Add caching logic here
    return await _remoteDataSource.getDashboardData();
  }
}
```

**Features**:
- ✅ Clean separation between domain and data layers
- ✅ Dependency injection via Riverpod providers
- ✅ Ready for future caching implementation
- ✅ Error propagation to presentation layer

## 🚀 Usage Guide

### Basic Usage

```dart
class DashboardScreen extends ConsumerStatefulWidget {
  @override
  ConsumerState<DashboardScreen> createState() => _DashboardScreenState();
}

class _DashboardScreenState extends ConsumerState<DashboardScreen> {
  @override
  void initState() {
    super.initState();
    // Fetch data when screen loads
    WidgetsBinding.instance.addPostFrameCallback((_) {
      ref.read(dashboardProvider.notifier).fetchDashboardData();
    });
  }

  @override
  Widget build(BuildContext context) {
    final state = ref.watch(dashboardProvider);
    
    return Scaffold(
      body: switch (state.status) {
        DashboardStatus.loading => Center(child: CircularProgressIndicator()),
        DashboardStatus.error => ErrorWidget(state.errorMessage),
        DashboardStatus.success => DashboardContent(state.data!),
        _ => EmptyState(),
      },
    );
  }
}
```

### Using Helper Providers (Cleaner Approach)

```dart
@override
Widget build(BuildContext context, WidgetRef ref) {
  final isLoading = ref.watch(isDashboardLoadingProvider);
  final error = ref.watch(dashboardErrorProvider);
  final data = ref.watch(dashboardDataProvider);
  
  if (isLoading) return LoadingWidget();
  if (error != null) return ErrorWidget(error);
  if (data == null) return EmptyWidget();
  
  return DashboardContent(data);
}
```

### Pull-to-Refresh

```dart
RefreshIndicator(
  onRefresh: () async {
    await ref.read(dashboardProvider.notifier).refresh();
  },
  child: YourListView(),
)
```

### Manual Refresh Button

```dart
IconButton(
  icon: Icon(Icons.refresh),
  onPressed: () {
    ref.read(dashboardProvider.notifier).fetchDashboardData();
  },
)
```

### Listening to State Changes

```dart
ref.listen(dashboardProvider, (previous, next) {
  if (next.status == DashboardStatus.error) {
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(content: Text(next.errorMessage ?? 'Error')),
    );
  }
});
```

## 🔧 Error Handling

The implementation uses your existing error handling infrastructure:

### Error Types (from `core/error/failure.dart`)

- **NetworkFailure**: No internet connection
- **ServerFailure**: Server errors (4xx, 5xx)
- **AuthFailure**: Authentication errors (401, 403)
- **UnknownFailure**: Unexpected errors

### Error Flow

1. **DioException** thrown by Dio
2. Caught by **DashboardRemoteDataSource**
3. Converted to **Failure** using `ErrorHandle.handleDioException`
4. Propagated through **Repository**
5. Caught by **Provider** and stored in state
6. Displayed in **UI**

## ⚡ Best Practices

### ✅ DO

- Use `ref.read()` for one-time reads (e.g., button clicks)
- Use `ref.watch()` for reactivity (rebuilds on changes)
- Use `ref.listen()` for side effects (snackbars, navigation)
- Handle all state cases in UI (initial, loading, success, error)
- Use RefreshIndicator for pull-to-refresh

### ❌ DON'T

- Call `fetchDashboardData()` in build method
- Use `ref.watch()` inside callbacks
- Ignore error states
- Mutate state directly

## 🆘 Troubleshooting

### Build Runner Issues

If you get errors about missing `.g.dart` files:

```bash
dart run build_runner build --delete-conflicting-outputs
```

### Provider Not Found

Make sure you've imported the generated file:

```dart
part 'dashboard_provider.g.dart';
```

### State Not Updating

Check that you're using `ref.watch()` in build method, not `ref.read()`.
